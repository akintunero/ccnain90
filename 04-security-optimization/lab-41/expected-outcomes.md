## Expected Outcomes – Lab 36 Security and Monitoring

After completing this lab, learners should be able to:

1. **Translate security policy into ACLs**
   - Define ACLs that selectively permit and deny traffic between branch, HQ, and external networks.
   - Place ACLs on appropriate interfaces and directions without breaking legitimate connectivity.
2. **Harden management access**
   - Restrict administrative access to dedicated management ranges.
   - Prefer SSH and secure authentication over insecure methods.
3. **Implement logging and basic monitoring**
   - Configure network devices to send logs to a central syslog server.
   - Interpret key log entries relating to ACL matches and security events.
4. **Evaluate the impact of security controls**
   - Observe how ACL and logging changes affect user experience and operational visibility.

These skills provide the security and monitoring baseline for more advanced optimisation and incident response labs.
